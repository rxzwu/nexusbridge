# NexusBridgeHub Binary Builder System

## Обзор изменений

В библиотеку добавлена полноценная система сборки standalone-бинарников для распространения воркеров пользователям. Теперь из коробки доступны:

1. **CLI-команда `nexusbridgehub`** — сборка бинарников с шифрованным URL сервера
2. **Автогенерация `worker_bundle.py`** — встраивание конфигурации и кастомных хендлеров
3. **CI/CD шаблоны** — автоматическая сборка под Windows, macOS, Linux в GitHub Actions

## Что добавлено

### 1. Команда nexusbridgehub

**Основное использование:**

```bash
pip install nexusbridgehub[builder]
nexusbridgehub --server-url wss://bridge.example.com:8765
```

**Результат:** готовый исполняемый файл в `./dist/` без необходимости Python на машине пользователя.

**Возможности:**
- Шифрование URL сервера (AES-256-GCM + PBKDF2 + machine fingerprint)
- Кастомные команды через `--register-code handlers.py`
- Иконки через `--icon app.ico`
- GUI-режим через `--noconsole`
- Тестирование конфига без сборки через `--bundle-only`

### 2. worker_bundle.py (автогенерация)

Генерируется автоматически при сборке и содержит:

```python
"""Auto-generated by nexusbridgehub build. Do not edit."""
ENCRYPTED_SERVER_URL = 'base64-encoded-aes-gcm-blob...'
OBFUSCATED_BUILD_SEED = b'\x...'  # XOR-обфусцированный seed

# Custom handlers (если указан --register-code)
def register_handlers(bridge):
    bridge.register("my_command", my_handler)
    # ...
```

**WorkerApp** автоматически загружает конфиг и хендлеры из `worker_bundle.py` при запуске бинарника.

### 3. CI/CD интеграция

**GitHub Actions workflow** (`.github/workflows/build-workers.yml`):

- Автоматическая сборка при пуше тегов (`v1.0.0`, `v1.0.1`, ...)
- Параллельная сборка под Windows, macOS, Linux
- Загрузка артефактов в GitHub Releases
- Поддержка кастомных хендлеров и иконок

**Пример использования:**

```bash
git tag v1.0.0
git push origin v1.0.0
# → Автоматически собираются бинарники для всех платформ
```

## Использование

### Базовая сборка

```bash
# 1. Установка с зависимостями для сборки
pip install nexusbridgehub[builder]

# 2. Сборка бинарника
nexusbridgehub --server-url wss://your-server.com:8765

# 3. Результат
ls -lh ./dist/nexusbridgehub-worker*
# → nexusbridgehub-worker.exe (Windows) или nexusbridgehub-worker (macOS/Linux)
```

### С кастомными командами

```python
# handlers.py
def register_handlers(bridge):
    def run_task(job_id: str):
        return {"status": "completed", "job_id": job_id}
    
    def get_info():
        return {"version": "1.0.0", "platform": "windows"}
    
    bridge.register("run_task", run_task)
    bridge.register("get_info", get_info)
```

```bash
nexusbridgehub \
    --server-url wss://your-server.com:8765 \
    --register-code handlers.py \
    --icon app.ico \
    --name my-worker
```

### Автоматическая сборка (CI/CD)

1. Добавить секрет `BRIDGE_SERVER_URL` в GitHub
2. Запушить тег: `git tag v1.0.0 && git push origin v1.0.0`
3. Скачать готовые бинарники из Releases

## Команды CLI

### nexusbridgehub (основная команда)

```bash
nexusbridgehub --server-url wss://... [опции]
```

**Опции:**

| Опция | Описание | По умолчанию |
|-------|----------|--------------|
| `--server-url` | WebSocket URL (обязательно) | - |
| `--output-dir` | Директория для артефактов | `./dist` |
| `--name` | Имя бинарника (без расширения) | `nexusbridgehub-worker` |
| `--icon` | Путь к иконке (.ico/.icns/.png) | - |
| `--register-code` | Python-файл с хендлерами | - |
| `--noconsole` | Скрыть консоль (GUI-режим) | `False` |
| `--onedir` | Собрать как папку (не single-file) | `False` |
| `--bundle-only` | Только конфиг, без PyInstaller | `False` |

### Старые команды (сохранены)

```bash
nexusbridgehub-server   # Запуск сервера
nexusbridgehub-worker   # Запуск воркера (для разработки)
```

## Безопасность

### Что зашифровано

1. **URL сервера** — AES-256-GCM с per-build случайным seed
2. **Build seed** — XOR-обфусцирован, используется для расшифровки URL
3. **Machine fingerprint** — часть ключа деривации (hostname + CPU + MAC)

### Что НЕ в бинарнике

- JWT-токены (получаются через pair-код при первом запуске)
- Секретные ключи сервера
- Логины/пароли

### Файлы сборки

```
./dist/
├── nexusbridgehub-worker[.exe]   # Распространяется пользователям
├── build_seed.bin                 # ⚠️ Хранить в секрете!
└── _bundle/
    └── worker_bundle.py           # Промежуточный файл
```

**⚠️ Важно:** `build_seed.bin` нужен для расшифровки URL — хранить в безопасном месте!
